## 2. 成本模型（腾讯云）

### 2.1 固定成本（月）

| 项目 | 保守 | 备注 |
|------|------|------|
| 腾讯云轻量 2C4G（广州/上海） | ¥45–65 | 含一定带宽 |
| COS 标准存储 50GB 内 | ¥1–5 | 按量 |
| CDN 流量 100GB 内 | ¥15–25 | 备案后启用 |
| 域名 .cn × 2 | ¥6（摊销） | devimage.cn + cdn 子域 |
| SSL 证书 | ¥0 | 腾讯云免费 DV 证书 |
| 云监控 | ¥0 | 基础免费 |
| **合计固定** | **¥67–101/月** | |

### 2.2 变量成本（随流量）

| 假设 | 单请求成本估算 |
|------|----------------|
| SVG 合成（无存储） | ~¥0.000001（可忽略） |
| WebP（Sharp 实时） | ~¥0.00001 |
| 缓存照片 100KB | COS 存储 ~¥0.000001/次 + CDN 流量 ~¥0.00001/次 |

## 3. 单位经济学

```
单用户 Free 档边际成本 ≈ 0（SVG 为主）
每 100 万次 SVG 请求 ≈ 50MB 出站 ≈ ¥0.04 流量费

Pro 用户 ¥49/月，配额 500 万：
  - 若全 SVG：流量成本 < ¥1
  - 毛利率 > 95%（无人力摊销）

Team 用户若大量使用 /photo：
  - 需监控人均流量，必要时降配额或加价
```

## 6. 收入路径优先级

```
1. 先 Free 获客 + 文档 SEO（掘金、GitHub README）
2. 3 个月后开 Pro（用量面板就绪后）
3. 6 个月后接 1–2 单 B2B 私有化
4. **12 个月后评估** COS 生命周期 + CDN 流量包优化
```

## 7. 关键财务指标（跟踪）

| 指标 | 公式 | 健康线 |
|------|------|--------|
| 成本/请求 | 月总成本 / 月请求 | < ¥0.0001 |
| 免费→付费转化 | 付费数 / MAU | > 1% |
| 带宽/请求 | GB / 百万请求 | < 0.05 GB（SVG 为主） |
| 毛利率 | (收入-变动成本)/收入 | > 80% |

## 9. 决策建议

1. **MVP 阶段全部免费**，只加 IP 限流，验证需求  
2. **Pro 在第一版用量统计上线后再开**，避免空承诺  
3. **真实照片 `/photo` 不放 Free 无限额度**，防止带宽打穿  
4. **B2B 比 C 端订阅更早产生 meaningful 收入**
