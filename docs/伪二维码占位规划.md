# 伪二维码占位规划

> **定位**：与占位图、头像、场景图并列的第四类 **图即资源**——提供「QR 视觉」的确定性 SVG/栅格占位，**不可扫描、不编码真实数据**。  
> **状态**：规划阶段；头像 Lab 已提供实验风格 `devimg-matrix` 用于算法与审美验证。

---

## 1. 背景与场景

| 场景 | 说明 |
| ------ | ------ |
| 支付 / 登录 mock | UI 需要二维码形占位，但不应误导用户去扫 |
| 列表骨架屏 | 小程序、App 卡片中的「码」形 loading |
| 技术风文档 | 开发者文档、API 示例中的装饰性矩阵 |
| 与头像区分 | 头像圆形/角色；伪 QR 强调 **方形模块阵 + 定位符** |

竞品对照：无统一国内 CDN 提供「假 QR 占位」；设计稿常用灰色方块或静态 PNG。

---

## 2. 产品边界

### 2.1 做什么

- seed 确定性生成 **类 QR 视觉**（定位符 + 数据区模块）
- 支持正方形尺寸（与头像类似 10–4000）
- 可选 **圆形裁剪**（query `shape=circle`，用于圆形头像位）
- SVG 默认；WebP/PNG 栅格与现有头像一致
- `Cache-Control: immutable`（固定 seed URL 可 CDN）

### 2.2 不做什么

- **不**生成可扫描的真实 QR（无 vCard、URL、WiFi 等编码）
- **不**对接扫码 SDK 或 zxing
- 文档与响应头明确标注 `X-DevImage-Pseudo-QR: 1`（规划）

---

## 3. 路由草案

与 [竞品 URL 与 API 对照表](./竞品URL与API对照表.md) 风格一致，独立根路径（不加 `/v1`）：

| 优先级 | 路由 | 说明 |
| ------ | ------ | ------ |
| P1 | `GET /qr/:seed/:size` | 正方形伪 QR，默认 SVG |
| P1 | `GET /qr/:seed/:size.webp` / `.png` | 栅格输出 |
| P2 | `GET /qr/:seed/:w/:h` | 矩形（非正方形时模块拉伸或留边） |
| P2 | `?fg=&bg=&accent=` | hex 配色 |
| P3 | `GET /qr/styles` | 多种视觉变体（矩阵密度、圆角模块、无定位符等） |

**与头像实验风格关系**：

- `devimg-matrix`：头像路由内的圆形/方形裁剪实验
- `/qr/*`：独立资源，文档与 Playground 单独一页 `apps/docs/api/qr.md`

## 4. 技术复用

```text
apps/api/src/
├── avatar/native-renderers/matrix.renderer.ts   # 已有：21×21 模块 + 定位符
└── qr/                                          # 规划：独立 Module
    ├── qr.module.ts
    ├── qr.controller.ts
    ├── qr.service.ts          # 复用/抽取 matrix 生成逻辑
    └── qr.service.spec.ts
```

抽取建议：

- 将 `renderMatrix` 核心抽到 `common/qr-matrix.ts` 或 `packages/` 共享
- 头像版：100×100 viewBox + 可选 clip
- 占位版：任意 `size` 或 `w×h`，模块计数可随尺寸缩放（21 / 25 / 29）

## 6. 文档与合规文案

用户文档须包含：

- 「生成的图案**不是**有效二维码，请勿用于支付或登录」
- 示例仅使用 `demo`、`placeholder` 等 seed
- 与文档站「使用规范与公平使用」（`apps/docs/guide/fair-use.md`）一致：固定 seed 推荐 CDN；随机 seed 仅演示

## 7. 验收标准（独立 `/qr` 上线时）

- [ ] 同一 seed 输出 deterministic
- [ ] 至少 3 个主流扫码 App 确认**无法**识别（抽检）
- [ ] 21×21 与 128×128 栅格清晰无锯齿
- [ ] 单元测试覆盖模块布局与定位符区域
- [ ] OpenAPI 登记 `/qr` 路由
