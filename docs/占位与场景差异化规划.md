## 2. 竞品差距分析

### 2.1 合成占位图 vs placehold.co

| 能力 | placehold.co | DevImage 现状 | 策略 |
| ------ | ------------- | --------------- | ------ |
| `/:w/:h` + text/bg/fg | ✅ | ✅ | 保持 |
| WebP / PNG | ✅ 全格式 | ✅ 已实现 | 补文档同步 |
| `800x600` 别名 | ✅ | ❌ | **P1 parity** |
| 路径配色 `/w/h/bg/fg` | ✅ | ❌ | **P1 parity** |
| border / cross | ✅ | ❌ | P2 |
| 字体 / Retina / AVIF | ✅ | ❌ | P3，非核心 |
| **73 分类真实图** | ❌ | ❌ | **P0 差异** |
| seed + 纹理/渐变 | ❌ | ❌ 头像 pattern 可复用 | **P1 差异** |

**结论**：灰块占位只需补迁移项达到 parity；**竞争力来自图库与平台联动**，不在灰块本身。

### 2.2 场景 / 状态 vs 组件库

| 能力 | Ant Design Empty 等 | DevImage 现状 | 一期策略 |
| ------ | --------------------- | --------------- | ---------- |
| SVG 插画 | ✅ | ❌ 纯文字 | **二期**（需设计/AI 部件） |
| URL 即用 | ❌ 组件依赖 | ✅ | 保持 |
| 骨架屏 CDN | ❌ | ❌ | **一期 P0**（纯程序） |
| theme / 自定义文案 | 部分 | ❌ | **一期 P1**（程序改色/改字，无插画） |
| seed 确定性 | ❌ | ❌ | 一期 placeholder/photo；场景 seed **随二期** |

**结论**：一期 **不做** 插画级场景升级（可灵试产未达验收）；
用 **骨架屏 + 文案 scene 增强 + 真实图库** 建立差异。部件化路线保留为二期，见 §6。

## 6. 场景插画：可行性分析（二期储备，一期不实施）

> **状态**：📋 **暂缓** — 可灵 AI 试产未达 Sprite 部件验收；一期不阻塞于此。  
> 以下分析保留为 **二期技术路线**；实施前提见 [场景部件交付规范](./场景部件交付规范.md)。

### 6.1 三种路线对比

| 路线 | 描述 | 设计投入 | 组合数 | 观感上限 | 推荐 |
| ------ | ------ | ---------- | -------- | ---------- | ------ |
| **A. 固定整图** | 每 variant 1–2 张 SVG/PNG | 高（每增场景需新稿） | ≈ 场景数 | ★★★★☆ | ❌ 非主方案 |
| **B. 纯程序绘制** | 类似 `geo.renderer` 参数化几何 | 极低 | 无限 seed | ★★★☆☆ | ✅ 骨架/icon |
| **C. 部件组装** | 背景 + 剪影 + 道具 + 文案 | 中（一次性部件库） | 笛卡尔积 | ★★★★☆ | ✅ **二期主方案** |

### 6.2 纯程序自动生成：能做什么、不能做什么

#### 6.2.1 高可行性（建议 100% 代码生成）

| 类型 | 实现方式 | 参考现有代码 | 可行性 |
| ------ | ---------- | -------------- | -------- |
| **骨架屏** | 圆角矩形 + shimmer 渐变 | 新建 `skeleton.renderer.ts` | **95%** |
| **背景层** | mesh / gradient / pattern | `native-avatar.service` 已有 | **95%** |
| **几何道具** | 空盒子、断链、放大镜、WiFi 斜杠 | `geo.renderer` 圆弧/路径 | **80%** |
| **布局框架** | 上插画下文案 / 左右分栏 | SVG `<g transform>` | **95%** |
| **主题色** | seed → HSL palette | `seedToHue` / `hslToHex` | **95%** |

#### 6.2.2 中可行性（程序 + 少量 SVG path 模板）

| 类型 | 说明 | 可行性 |
| ------ | ------ | -------- |
| 简化线性人物 | 3–5 个参数化 stick figure / blob 人形 | **70%** |
| 场景隐喻 icon | 空文件夹、购物车、聊天气泡（单 path 图标） | **75%** |
| 装饰粒子 | 问号、省略号、小星星（seed 散布） | **85%** |

#### 6.2.3 低可行性（不建议程序硬做）

| 类型 | 原因 |
| ------ | ------ |
| 精细角色插画（unDraw 级） | 需大量贝塞尔手工或 AI 出图，程序难以稳定产出 |
| 强叙事场景（多人互动、透视建筑） | 组合爆炸，代码维护成本 > 设计成本 |
| 与品牌强绑定的营销风插画 | 超出占位 CDN 范围 |

**观感预期**：纯程序路线可达到 **「现代极简 empty state」**（类似 Stripe / Linear 几何风），
**达不到** Ant Design 内置插画的细腻度 — 但对 **开发者占位** 场景足够，
且 seed 变体是组件库没有的。

### 6.3 固定十几套整图的问题

| 问题 | 说明 |
| ------ | ------ |
| **无 seed 差异** | 全站 empty 页长得一样，Demo 不真实 |
| **场景覆盖不足** | 404/empty/network 只是起点；电商、IM、权限、维护等需 15+ 场景 |
| **线性成本** | 每增 1 场景 ≈ 1 张新插画，10 场景后设计成为瓶颈 |
| **尺寸适配差** | 固定 viewBox 在 375×200 与 1200×600 难共用 |
| **与平台割裂** | 无法与 avatar/placeholder 共享 palette、pattern |

因此：**固定整图仅适合 MVP 验收 2–3 张**，不应作为长期架构。

### 6.4 推荐方案：Scene Composer（部件化组装）

类比头像体系：

```text
头像（已有）                          场景（规划）
─────────────────────────────────────────────────────────
style registry                    →   variant recipe 表
native renderers (geo, face…)     →   part renderers (bg, prop, figure)
devimg-patterns                   →   scene accents / 装饰
seed → 确定性选件                 →   seed → 确定性选件
GET /avatar/styles                →   GET /scene/styles
manifest (devimage-cn 规划)       →   scene-parts manifest（可选 PNG 层）
```

#### 6.4.1 部件槽位（Scene Slots）

| 槽位 | 数量建议 | 来源 | 说明 |
| ------ | ---------- | ------ | ------ |
| **background** | 6–8 | 代码生成 | mesh / gradient / 轻纹理，与 avatar 共用逻辑 |
| **figure** | 8–12 | 程序 blob **或** 极简 PNG 剪影 | 侧影/背影即可，无需五官精细 |
| **prop** | 16–24 | **程序 SVG 为主** | 盒子、购物车、文档、搜索、信号塔、锁、 wrench… |
| **accent** | 8–12 | 代码 | 问号、省略号、虚线、小图标 |
| **layout** | 3–4 | 代码 | stack / side / compact / wide |
| **copy** | 按 variant | 文案表 | 中文默认，query 可覆盖 |

**组合估算**：8 × 10 × 20 × 4 ≈ **6400** 种视觉组合（尚未计 accent 与 palette 变体）。  
部件数远低于「6400 张插画」，与头像 16³ = 4096 思路一致。

#### 6.4.2 Variant Recipe（场景 → 部件约束）

每个业务 variant 不是「一张图」，而是一条 **配方**，约束 prop 池与默认文案：

| variant | prop 池（示例） | 默认文案 |
| --------- | ----------------- | ---------- |
| `404` | 断链、迷路牌、问号地图 | 页面不存在 |
| `empty` | 空盒子、空文件夹、空托盘 | 暂无数据 |
| `network` | WiFi 斜杠、断线、信号塔 | 网络异常 |
| `search` | 放大镜、空结果列表 | 无搜索结果 |
| `cart` | 空购物车 | 购物车是空的 |
| `message` | 空气泡 | 暂无消息 |
| `permission` | 锁、盾牌 | 暂无权限 |
| `maintenance` | 扳手、施工锥 | 系统维护中 |

同一 `empty` variant，`seed=shop-a` 与 `seed=shop-b` 可选不同 prop/figure 组合，
**列表页 mock 不会撞图**。

#### 6.4.3 路由设计（草案）

```http
GET /scene/:variant
GET /scene/:variant/:seed
GET /scene/:variant?w=800&h=600&theme=dark&title=&subtitle=&accent=6366f1
GET /scene/styles
GET /skeleton/:w/:h?type=card|row|grid|page&theme=dark&animate=0
```

| 参数 | 说明 |
| ------ | ------ |
| `seed` | path 或 query；决定部件下标与 palette |
| `theme` | `light` \| `dark` |
| `title` / `subtitle` | 覆盖中文文案 |
| `accent` | 品牌色 hex |
| `layout` | 可选，覆盖 recipe 默认布局 |

#### 6.4.4 代码结构（草案）

```text
apps/api/src/scene/
├── scene.module.ts
├── scene.controller.ts
├── scene.service.ts              # 编排入口
├── scene-recipes.ts              # variant → prop 池 + 默认 copy
├── scene-composer.ts             # 按 seed 选件、算布局
├── parts/
│   ├── backgrounds.ts            # 程序生成
│   ├── props.ts                  # SVG path 图标库
│   ├── figures.ts                # blob / 剪影
│   └── accents.ts
├── skeleton/
│   ├── skeleton.controller.ts
│   └── skeleton.renderer.ts      # 100% 程序
└── scene-styles.catalog.ts       # GET /scene/styles
```

### 6.5 设计投入估算（部件方案 vs 整图方案）

| 项目 | 固定整图（15 场景） | 部件组装 |
| ------ | --------------------- | ---------- |
| 初始设计 | 15 张完整插画 ≈ 15–30 人日 | 20–30 个部件 ≈ 8–12 人日 |
| 新增 1 业务场景 | 1–2 人日（新插画） | **0.5 人日**（加 recipe + 选 prop 池） |
| 程序开发 | 低 | 中（2–3 周，可复用 avatar 模式） |
| seed 变体 | 无 | 有 |
| 长期维护 | 差 | **优** |

**可选加速**：prop / figure 层可用 **可灵 / AI 批量出极简 flat 剪影 Sprite**
（参考 [头像美术资源规范](./头像美术资源规范.md) 的 8×8 网格流程），
程序负责裁剪与 anchor 对齐 — 设计只审部件，不画整场景。

## 7. 骨架屏（一期 P0，100% 程序）

骨架屏 **不依赖** 设计素材，一期优先落地：

| 类型 | 路由示例 | 实现 |
| ------ | ---------- | ------ |
| 全页 | `/skeleton/375/812?type=page` | 顶栏 + 3 卡片条 |
| 卡片 | `/skeleton/350/120?type=card` | 左图右文 |
| 列表行 | `/skeleton/300/64?type=row` | 头像圆 + 两行 |
| 网格 | `/skeleton/800/600?type=grid&cols=3` | 等分 cell |
| 动画 | `?animate=1` | SVG gradient 动画或静态（三期 APNG） |

**竞争力**：国内无统一 CDN 提供 `<img src="...skeleton...">`，与场景模块形成 **「加载态 + 空状态」** 双子能力。

**竞争力**：国内少见 CDN 级 `<img src="...skeleton...">`；一期与 `/photo` 并列核心差异。
